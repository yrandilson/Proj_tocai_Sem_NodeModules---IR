# Guia de Documentacao - TrocaAi

Esta pasta foi reorganizada por area de estudo do sistema para facilitar manutencao, revisao e onboarding.

## Governanca de Atualizacao (obrigatorio)

- Processo oficial: `01_visao_geral/PROCESSO_ATUALIZACAO_DOCUMENTACAO.md`
- Template padrao para novos docs: `01_visao_geral/TEMPLATE_DOCUMENTO_PADRAO.md`
- Regra: toda mudanca relevante de codigo deve atualizar documentacao da area impactada no mesmo ciclo.

## Estrutura por estudo

- `01_visao_geral/`: leitura inicial, mapa mental, visao do sistema
- `02_arquitetura_design/`: arquitetura, diagramas e responsabilidades de arquivos
- `03_requisitos_conformidade/`: requisitos funcionais, conformidade e especificacao futura
- `04_backend_api_dados/`: endpoints, mapeamento de metodos e documentacao tecnica de backend
- `05_frontend_ux/`: reservado para estudos de frontend e experiencia (expansao futura)
- `06_testes_qualidade/`: estrategia de testes, casos de teste e conceitos de qualidade
- `07_operacao_setup_deploy/`: instalacao, export, scripts e guias operacionais
- `08_seguranca/`: melhorias e sumarios de seguranca
- `09_relatorios_revisoes/`: relatorios tecnicos e historico de revisoes/correcoes
- `10_apresentacoes/`: material de apresentacao
- `11_fluxos_uml/`: fluxos detalhados e simulacoes
- `99_arquivo_historico/`: copias antigas, rascunhos e arquivos preservados para rastreabilidade
